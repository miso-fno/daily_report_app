## 1. テスト概要

### 1.1 目的

本書は、営業日報システムの品質を保証するために実施するテストの仕様を定義する。

### 1.2 テスト範囲

| テスト種別     | 対象               | 目的                                     |
| -------------- | ------------------ | ---------------------------------------- |
| 単体テスト     | 各機能・API        | 個々の機能が正しく動作することを確認     |
| 結合テスト     | 機能間連携         | 複数機能の連携が正しく動作することを確認 |
| システムテスト | システム全体       | 要件を満たしていることを確認             |
| 受入テスト     | 業務シナリオ       | 実業務で利用可能であることを確認         |
| 非機能テスト   | 性能・セキュリティ | 非機能要件を満たしていることを確認       |

### 1.3 テスト完了基準

- 全テストケースの実施完了
- 重大度「高」「中」のバグが0件
- テストケース合格率95%以上
- 重大度「低」のバグが10件以下

### 1.4 重大度定義

| 重大度 | 定義                                     | 対応                   |
| ------ | ---------------------------------------- | ---------------------- |
| 高     | システムが停止する、データ破損が発生する | 即時対応必須           |
| 中     | 主要機能が使用できない、回避策がない     | リリース前に対応必須   |
| 低     | 軽微な不具合、回避策がある               | 次回リリースまでに対応 |
